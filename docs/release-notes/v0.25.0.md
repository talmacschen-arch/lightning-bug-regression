### 主要变化（design.md §0 v1.20 → v1.25）

- **M7 LLM 接入交付**（v1.25，PR #195/#196/#198）—— 新 endpoint `POST /cases/generate-draft`，用 Anthropic SDK（`anthropic>=0.40,<1`，模型钉 `claude-opus-4-7`）从自然语言描述生成 case YAML 草稿。要求 Bearer auth；`description ≤ 8KB` / `max_tokens=2000`；schema-invalid 重试 ≤2 次（把上次校验错误塞回 prompt），Anthropic API 错（429/5xx/超时）按错误码上抛不重试；prompt caching 打在 schema+few-shot 共同前缀。前端 `/cases/new` "从描述生成" 入口接线（key 缺失时禁用并提示）。

- **review 流水线补齐并端到端验证**（v1.22 → v1.24）—— reviewer 焊进 foreman 作 merge 前置闸门；smoke-runner 落地（merge 后跑，NO-GO 自动开 revert PR）；修复 smoke 终态门在 `claude --print` 一次性进程里被后台化导致 orphan + 丢 final JSON 的编排 bug（改前台同步派发）。v1.24 首次在真实「多 PR 功能 sprint」（m6-run-experience-deepening，6 PR #189~#194）上端到端跑通自动链，wrapper 对账 `r25_violation=False`。

- **运行体验深化 + 集群修复 wave**（v1.20）—— 同日 17 PR 集群：`run.errored` 列 + error verdict + 5-counter（passed/failed/skipped/errored/total）；`/runs/:id` 实时进度条；case 间隔离（`RESET ALL` + `DEALLOCATE ALL` + `prepare_threshold=None`，修 psycopg3 auto-prepare 与会话态泄漏）；时区 root fix（后端统一 tz-aware UTC，丢弃 `datetime.utcnow`，修 UTC+8 下 ETA 偏 ~8h）；admin 网关从 `X-Admin-Password` 迁移到 Bearer token；observability（step_id 路径字符消毒 + 驱动异常写 `.error.txt`）；`hadoop_simple` external cluster 配置。

- **external_systems status_whitelist 双轴改造**（v1.21）—— whitelist 改 `[open, fixed, wontfix, stub, awaiting_env]`，default `open`：主轴对齐 bug_regression 的 BUG 修复状态，`awaiting_env` 表达外部服务未部署占位（alembic 0006）。

- **新增 external_systems case**（×2）—— `lg-xs-pxf-hdfs-order-by-writable`（#175）、`lg-xs-pxf-hive-fdw-encoding-utf8`（#177）；`/cases/try` 对齐 `POST /runs` 加载 external_deps Jinja context（#174）。

### Breaking changes / Migration notes

- **`/cases/generate-draft` 需要 Bearer auth**（不沿用 `/cases/*` 其余 endpoint 的匿名姿态）；调用前先 `POST /login` 拿 token。
- **需要 `ANTHROPIC_API_KEY` 环境变量**才能用 LLM 草稿生成；缺失时 endpoint 返 503、前端入口禁用。这是新增可选依赖，不影响其余功能。
- 升级需 `alembic upgrade head`（本区间含 0005 `runs.errored` / 0006 external_systems whitelist）。fresh clone 跑 `bash scripts/bootstrap.sh`。

### 已知限制

- 单用户设计（admin/admin + bcrypt + 不过期 Bearer token）；多用户需 schema 重做。
- M7 LLM 草稿生成 polish backlog 未做：401/其他错误码区分、socks 代理、文案打磨（见 commit `03d625c`）。
- LLM 调用日志只 `logger.info` 落 tokens/latency/retry_count，不持久化。
