### Sprint M5-layout — Layout 重构（hard prerequisite，先做）

- [x] **M5-1 Layout 重构 sidebar**：`frontend/src/components/Layout.tsx` 从 top nav 改 **sidebar + main content** 两栏布局
  - **sidebar 内容**：Logo / 主导航（Dashboard / Cases / Runs / Admin disabled）/ 当前活跃 run 指示器（红/绿/黄 pip + tooltip "Run #N PASS 4h ago"，数据源 `GET /runs?limit=1`）
  - **main content**：breadcrumb（基于 react-router pathname）+ Outlet + footer (env / version 静态)
  - **响应式**：sidebar 宽度 240px 固定，< 1024px 自动 collapse 成 icon-only（不强求；M5 不做移动端）
  - **实施提示**：复用 shadcn/ui `Sidebar` + `Breadcrumb` 组件；React Router nested layout
  - **data-testid**：`sidebar-nav-dashboard` / `sidebar-nav-cases` / `sidebar-nav-runs` / `sidebar-active-run-pip` / `breadcrumb` / `main-content`
  - **测试**：vitest 验 sidebar 渲染 + active-run-pip 取数；Playwright contract test 验 click 跳转
  - 旧 Layout.tsx 的 3 个 top nav 链接全部替换；不保留兼容

### Sprint M5-dashboard — Dashboard `/` 落地页（M5 核心；依赖 M5-1）

- [x] **M5-2 Dashboard `/` 落地页**：根路径 `/` 不再 redirect `/cases`，新建 `DashboardPage.tsx` + `/dashboard` 落地 + `/` redirect 到 `/dashboard`
  - **4 个 KPI Card**（用 shadcn `Card`）：
    1. **Total cases by category** — 柱图，3 类（按 `GET /admin/categories` 渲染，不写死，§14 R4b）
    2. **Recent runs (7 天)** — 折线 + bar，pass/fail 比例，数据 `GET /runs?since=7d`
    3. **BUG status pie** — open / fixed / wontfix / stub，数据 `GET /cases?category=bug_regression` 客户端聚合
    4. **Extension stability** — stable / experimental / deprecated / stub / awaiting_env（external_systems 也含），多类堆叠（v1.21 修订：external_systems whitelist 已改为 `[open, fixed, wontfix, stub, awaiting_env]`，KPI tile 数据驱动按各 category whitelist 自动渲染，不再硬绑此处描述）
  - **Recent activity 列**：`GET /runs?limit=10`，每行 status badge + duration + click 跳 `/runs/:id`
  - **Quick actions** 横排：3 个常用 preset（Run all bug_regression open / Run all extension stable / View skip_list）；click → `/runs/new?preset=<name>` 预填
  - **数据源**：用现有 endpoint；如客户端聚合慢，本步**不**加 `GET /admin/stats`（M6 再考虑后端聚合），先走客户端聚合 + memoize
  - **图表库**：复用 recharts 或 shadcn ui 自带（先选轻量；不引入大依赖）
  - **data-testid**：`dashboard-card-categories` / `dashboard-card-recent-runs` / `dashboard-card-bug-pie` / `dashboard-card-ext-stability` / `dashboard-recent-activity` / `dashboard-quick-actions`
  - **测试**：vitest 用 mock data 验渲染；Playwright `page.route()` mock 4 endpoint 验 KPI 数字
  - **R4b 强校**：所有 category 来自 API，不写死

### Sprint M5-x-page-relations — 跨页关联 + 筛选 + Quick Actions（M5-1 之后并行）

- [x] **M5-3 跨页关联 backend + frontend**
  - **backend**：新 endpoint `GET /cases/:id/recent-runs`（`backend/app/api/cases.py` + `case_results` storage），返 最近 10 条 case_results JOIN runs，按 started_at desc。**§14 R26 强校**：复用 `case_results` storage 模块，禁止 inline SQL

  - **frontend 2**：RunDetailPage 每行 case 加 click 跳 `/cases/:id`（已有 case_id 字段）
  - **测试**：backend pytest test endpoint；frontend vitest mock 验区块渲染 + Playwright 验跳转
- [x] **M5-4 全局筛选 + URL 持久化**
  - 新 `frontend/src/components/FilterBar.tsx`：category multi-select（API 拉）/ status multi-select / tag autocomplete / time range picker
  - `/cases` + `/runs` 都用同一份 FilterBar 组件（替换现有 inline filter）
  - 筛选状态 → URL query string，刷新页保留；分享链接他人可见同视图
  - 实施提示：React Router `useSearchParams` + custom hook `useFilters(): {filters, setFilters}`
  - **data-testid**：`filter-bar` / `filter-category` / `filter-status` / `filter-tag` / `filter-time`
  - **R26 强校**：FilterBar 是单一来源，CasesPage / RunsPage 引用同一份；禁止 inline 复制
- [x] **M5-5 Quick Actions (RunNewPage 重做)**
  - `/runs/new` 当前是 "所有 case checkbox + 全选" 太 raw
  - 重做为：**preset 卡片区**（3 个常用：Run all bug_regression open / Run all extension stable / Run all by current filter）+ 下方原 case checkbox 区作为 "Advanced selection"
  - preset 卡片 click → 自动填 case 列表 + 跳 confirm 模态 + POST /runs
  - sidebar Quick Actions 下拉支持从任意页面触发（`?preset=<name>` URL）
  - **data-testid**：`runnew-preset-bug-open` / `runnew-preset-ext-stable` / `runnew-preset-filter` / `runnew-advanced`
  - **R4b 强校**：preset 卡片 label 来自 categories API 渲染，"bug_regression" 名字不写死
