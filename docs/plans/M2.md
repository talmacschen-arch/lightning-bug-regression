# M2 sprint — frontend MVP

foreman 入口文件（design.md §13.0-E）。M2 = 前端 MVP：把 M1 已就绪的 backend API 接出 UI，跑通 "看 case → 触发 run → 看结果" 端到端体验。

权威设计：`design.md`
- §6（前端目录 + 路由 + 选型）+ §6.3（Vite + React + TS strict + Tailwind + shadcn/ui + openapi-typescript）
- §6.4（前端强约束 8 条）— 必读
- §13.5（M2 子步骤计划，本 sprint 的来源）
- §5.2（已就绪 backend endpoint 表，M2 消费）

§14 预付教训 — 写代码 + reviewer cross-reference 必看：
- **R2**（contract test）：表单提交必有 `page.route("**/api/runs", ...)` + `postDataJSON()` 断言提交体 shape；可见性测试（"button 存在"）不算
- **R4b**（不硬编码 category）：所有 category 列表 / tab 渲染 / 状态颜色 必须从 `GET /admin/categories` 拉，禁止 `if category === "bug_regression"` 字面比较；codegen 类型 `category` 标 `string` 不是枚举
- **R6**（data-testid）：所有 E2E 待选元素必须 `data-testid="..."`；禁止 `.card + .pool-label` 结构选择器；禁止 `.first()` 依赖渲染顺序
- **R7**（ErrorBoundary）：根组件强制 `<ErrorBoundary>`；任何子树抛错必有降级 UI；禁止白屏
- **R8**（test.skip declaration-level）：`test.skip("name", fn)` 写在 declaration 不在 body 里
- **R24**（specialist 本地 ci-gate triplet）：frontend-fixer step 1 必须**真跑** `npx tsc --noEmit && npm run lint && npx vitest run`（playwright if touched）；脑内推断"应该绿"不算
- **R25**（foreman dispatch 走 wrapper）：调起 foreman session 一律 `scripts/dispatch-foreman.sh`，**不**直接 `claude --print --agent foreman`

reviewer 必须按 §14 cross-reference + §6.4 8 条逐项核对（详 `.claude/agents/reviewer.md` step 5）。

## 失控防护

按 design.md §15.4 + foreman.md hard rules。本 sprint 特别注意：
- frontend-fixer 第一次跑会触发**很多**新 npm 依赖装载 + ci-gate frontend block 首次跑——`npm install` 耗时 + 网络敏感，第一次给宽松 timeout；同 symptom hash fail 2 次（如 "npm install ETIMEDOUT" 重复）→ escalate
- **§14 R24 footgun 高发**：frontend-fixer step 1 必须**真跑** `npx tsc --noEmit + npm run lint + npx vitest run`；M1 backend 在 ruff format --check 上栽过 2 次 (PR #18 / #22)，前端不要重蹈
- 10 round 或 2h budget → BUDGET-EXHAUSTED；budget 大概率不够整 10 item，下一次 invocation 从本文件当前状态接力（M2b 4 路并行也许能在第二次 session 集中 burn 掉）
- 阻塞类（npm registry 不可达 / playwright browser download 失败 / vite dev server 起不来）写 needs_human 不重试
- M2-10 dogfood 是**人类**手动跑（非 foreman 自动），foreman 把前 9 项做完即 status=done，M2-10 留 needs_human entry "等用户开浏览器验"
