## 任务

- [ ] **m7-backend**（backend-fixer）: 见 design.md §13.13 (含 v1.25 amendment) §5.4。包含：

  **① M7-prep**:
  - `anthropic` 加进 `backend/pyproject.toml` deps（当前未装，已核实 2026-05-28）
  - `README.md` "起本机 dev 服务" 必需 env 表加 `ANTHROPIC_API_KEY`

  **② M7-1 endpoint `POST /cases/generate-draft`**（`backend/app/api/cases.py`）:
  - **Bearer auth 必须**（A）：`dependencies=[Depends(get_current_user)]`，对齐 `/runs` + `/admin/*` 当前 v1.17+ 姿态。**不**沿用 `/cases/{validate,try,submit}` 当前匿名姿态（独立议题，本 sprint 不动那三个）。
  - request: `{description: str, category?: str | None}`；**`len(description) > 8 KB` → 413**（B）
  - response: `{yaml_draft: str, attempts: int, validation_errors_during_retry: list[str]}`
  - Anthropic SDK `max_tokens=2000`（B），model=`claude-opus-4-7`（D1）
  - 校验**复用** `yaml_loader.parse + case_normalizer.normalize_case`（§14 R26），**禁** inline 校验
  - **retry 分两类**（C）：
    - schema-invalid → retry ≤2 次，**上次的 validation error 串塞回下次 prompt**
    - Anthropic API 错（429 / 5xx / 网络 / 超时）→ **不重试、不进 retry quota**，按错误码上抛
  - **错误码细分**：503 (key 缺) / 502 (Anthropic 5xx) / 429 透传 / 504 (超时) / 413 (description 超长) / 401 (no Bearer) / 200 + body attempts=3 (2 次 schema retry 失败，业务态非错)
  - **可观测性**（F）：每次调用 `logger.info` 落 `prompt_tokens / completion_tokens / latency_ms / retry_count / validation_errors_count / model`，不持久化

  **③ M7-2 prompt module**（新建 `backend/app/api/llm_prompt.py`）:
  - System prompt 内容：§4.1 schema 简版 + canonical 字段顺序（§17.6）+ status_whitelist（按 category 注入，`{{ALLOWED_CATEGORIES}}` placeholder fetch from `/admin/categories` at request time，§14 R4b 不硬编码）
  - **(E-1)** 显式写"未指定 database 时填 `gpadmin`"（2026-05-24 新规范 [[default_database_gpadmin]]，不只靠 few-shot 推断）
  - **(E-2)** 显式写 §4.1.2 psql -c 铁律 —— non-tx-safe DDL（VACUUM / REINDEX CONCURRENTLY / CREATE DATABASE / ANALYZE 顶层）**绝禁** `kind: sql`，**必须** `kind: shell + cmd: psql -c '<DDL>'`
  - 3 个 hardcoded few-shot：`lg-bug-0001-hashjoin-right-table`（BUG）/ `lg-ext-pgvector-ivfflat-basic`（extension）/ `lg-bug-0008-pax-toast-vacuum-analyze-crash`（cross-driver §4.1.2）
  - **Prompt caching**（G）：schema + few-shot **共同前缀**打 `cache_control: {"type": "ephemeral"}` marker；user `description` **不**缓存。5-min TTL 在低流量场景命中率有限，定位"省点是点"非承诺

  **④ backend pytest**（同 PR inline，§14 R26/R27 wiring）:
  - mock `anthropic.Anthropic.messages.create` happy path（返合法 YAML → 200 + yaml_draft）
  - retry-on-invalid 链路（mock 第 1 次返恶意 YAML / 第 2 次返合法）→ 200 attempts=2
  - **(D 必 assert wiring)** 第 1 次失败后，断言第 2 次 `messages.create` 调用的 prompt body **真包含上次 validation error 串**
  - 2 次都返恶意 YAML → 200 attempts=3 + validation_errors_during_retry 非空
  - Anthropic API 错（mock 429/5xx/超时）→ **不进 retry**，按错误码（429/502/504）上抛
  - 无 `ANTHROPIC_API_KEY` env → 503 + body 明示
  - description > 8KB → 413
  - **(A 契约测)** 无 Bearer auth → 401

  **Out of scope（本项不做）**：碰前端、改 `/cases/{validate,try,submit}` 的 auth、给 endpoint 加 streaming/multi-turn/Regenerate 按钮、错误文案精细化。
